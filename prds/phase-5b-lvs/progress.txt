## Codebase Patterns
- LVS follows DRCRunner pattern: same __init__(binary, timeout), sync run(), async async_run()
- CPU throttling uses triple-layer: taskpolicy -b (macOS) + nice + cpulimit
- PDKConfig fields are optional via `| None` with `default=None` — no need to update existing pdk.json files
- Test fixtures use tmp_path for PDK dirs + patch PDK_CONFIGS_DIR
- The .lvsdb format is NOT XML — it's a custom S-expression-like text format starting with `#%lvsdb-klayout`
- .lvsdb has three sections: layout(...), reference(...), xref(...) with match/nomatch/mismatch statuses
- Short-form keywords exist (J=layout, H=reference, Z=xref, X=circuit/nomatch, D=device, N=net) — parser handles both
