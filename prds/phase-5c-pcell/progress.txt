## Codebase Patterns
- Use gdstk.rectangle for all polygon construction — no boolean ops needed for basic shapes
- SKY130 layer numbers defined in pdk.json; use constants (LYR_DIFF, LYR_POLY, etc.) in generators
- PCellGenerator.snap_to_grid(value, 0.005) for all coordinates — SKY130 grid is 5nm
- Use unified contact pitch (max of licon and mcon spacing) to avoid spacing violations
- T-gate widening needed when gate L < 0.270um (licon + 2*enclosure)
- Cell names: sky130_{device}_{params} with 'p' for decimal points
- RPM layer (86/20): min width 1.27um, min spacing 0.84um — much larger than poly min width
- poly_rs (66/13): marks resistive region on poly
- Precision resistor = RPM & PSDM & poly.interacting(poly_rs) — all three layers needed
- NPC (95/20) layer needed for poly-to-contact transitions
- _place_terminal_contacts helper can be reused for any poly-based terminal connection
- _via_array(x0, y0, x1, y1, size, spacing, margin) — reusable 2D via/contact array placer
- CAPM layer (89/44) not in pdk.json — hardcoded in capacitor generator
- Met3 enclosure of CAPM (0.14um) too narrow for via2 — must extend met3 with landing pad
- API routes: register static paths (/devices) before dynamic paths (/{id}/download) to avoid FastAPI conflicts
- DRCReport fields: description, original_file, generator, top_cell — NOT source_file or total_violations as constructor arg
